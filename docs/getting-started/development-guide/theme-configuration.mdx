# Theme Configuration

The Infermedica Component Library is a highly configurable UI components library. We provide our default design system, which allows almost every design to be adjusted.
To configure the Infermedica Component Library to your needs, you can use a lot of CSS variables and adjust the styles to your requirements.
Below you'll find information about how to configure the different aspects of the styles and components.

<PageOutline/>

## CSS custom properties (--*): CSS variables
CSS custom properties (also called CSS variables) allows for the creation of custom properties that contain a value that can be used in other declarations by using `var()`.
CSS variables is similar to SASS and LESS variables but it's a native CSS feature. It doesn't need to be compiled like SASS or LESS.
It can also be overridden by inline styles and js part. E.g. we can calculate some values in the `computed` function and then use them in inline styles.

##  Configuring the CSS
CSS is a powerful tool to configure the look and feel of your project. It usually covers things like colors, fonts, and other visual elements.
The Infermedica Component Library uses a few different levels to configure the CSS of your project.

### Global custom properties
Global variables represent the project's style guide. They are based on the Figma project where we keep our Style Library, with tokens for color, font, font styles, space, border-radius and more.
Our tokens use the `html` scope to set up the default values e.g.

<Code code={`html {
  --space-2: 0.125rem;
  --space-4: 0.25rem;\n
  --border-radius-container: 5px;
  --border-radius-button: 5px;
  --border-radius-form: 5px;
}`}/>

`html` has lower priority than the `:root` scope so you can override the default values by adding your own custom properties to the `:root` scope e.g.

<Code code={`:root {
  --border-radius-container: 0;
  --border-radius-button: 0;
  --border-radius-form: 0;
}`}/>

RTL is supported by adding the `:root` scope using the `dir: rtl` property e.g.

<Code code={`:root {
  --font-family-heading: "Roboto", sans-serif;
  --font-family-body: "Roboto", sans-serif;
}\n
:root[dir="rtl"] {
  --font-family-heading: "IBM Plex Sans Arabic", sans-serif;
  --font-family-body: "IBM Plex Sans Arabic", sans-serif;
}`}/>

### Themes
For styles like `secondary` or `brand`, we use themes that follow the BEM convention. A theme is a type of modifier that has `-theme-` in the name. e.g.
`ui-button--theme-secondary`.

To set a style for the theme, we override the values of the default variables used in the component. e.g.

<Code code={`.ui-button {
  @at-root [class*="-secondary"] {
    #{$this},
    &#{$this} {
      --color-text-action-primary: var(--color-text-action-secondary);
      --color-text-action-primary-hover: var(--color-text-action-secondary-hover);
      --color-text-action-primary-active: var(--color-text-action-secondary-active);
      --color-icon-primary: var(--color-icon-secondary);
      --color-icon-primary-hover: var(--color-icon-secondary-hover);
      --color-icon-primary-active: var(--color-icon-secondary-active);
    }
  }
}`}/>

Using the selector allows users to set a theme for the component. e.g.

```html
 <UiButton class="ui-button--theme-secondary">Label</UiButton>
```

or do a scope, where all of the `UiButtons` are affected by the theme. e.g.

```html
  <UiNavigation class="ui-navigation--theme-secondary">
    <UiNavigationItem>Label</UiNavigationItem>
    <UiNavigationItem>Label</UiNavigationItem>
  </UiNavigation>
```

`ui-navigation--theme-secondary` was not defined in the `UiNavigation` component but `UiButton` still supports this approach, thanks to the `[class*="-secondary"]` selector.

You can create your own themes e.g. a dark or light theme.

### Component-specific custom properties
Global variables cover many cases. But sometimes, you may need to configure the CSS of a specific component like a button or icon (e.g. the icon's size), ...
For example, you may want to change the background color of every `UiButton` component.

<Code code={`.ui-button {
  --button-background: #1471C9;
  --button-hover-background: #0F5496;
  --button-active-background: #0B3D6D;
}`}/>

or the size of the icon

<Code code={`.ui-icon {
  --icon-size: 1.5rem;
}`}/>

Component specific variables are defined for configurable properties like colors, backgrounds, borders, paddings, margins, etc.
The list of CSS variables for each component is available in the component's documentation.

#### BEM and CSS custom properties
We use the BEM naming convention for CSS variables. In common cases, it will be `--<block>-<element>-<modifier>` e.g.

<Code code={`.ui-side-panel {
  &__content {
   padding: var(--side-panel-content-padding, var(--space-24) var(--space-20));
  }
}`}/>

But in some cases, that's not enough.

<Code code={`//states (hover, active)
  --button-icon-color: var(--color-icon-on-action);
  --button-hover-icon-color: var(--color-icon-on-action-hover); // state will be added after the block/element name that used that state i.e. hover is also used for button, not only for icon\n
  //checked state
  --checkbox-checked-background: var(--color-selectioncontrols-selection);
  --checkbox-checked-hover-background: var(--color-selectioncontrols-selection-hover); // some components have a checked state, which will be added before the state because checked is more static than temporary states like hover.`}/>

CSS variables only covers the design cases - i.e. breakpoints, rtl, states - when it is added to the required components. That way, when you need to cover a new case, e.g. a new breakpoint, you can do it.

<Code code={`@use "@infermedica/component-library/styles/mixins";'\n
.ui-checkbox {
  &__checkmark {
    @include mixins.for-desktop {
      --icon-size: var(--checkbox-desktop-checkmark-size, 1.5rem);
    }
  }
}`}/>

You can get more information about CSS variables and our approach in the [CSS Variables Deep Diving](docs/contributing-guide-css-var-deep-dive--page) section.

### Per-instance custom properties
Sometimes you may need to configure the CSS of a specific instance of a component.
For example, you may want to change the background color of a specific `UiButton` component.

This is the recommended approach to configuring the CSS of a specific component as it's easier to maintain.
Instead of overriding the property of e.g. `.ui-button`, you create a new class and add it in the required places. E.g. You can create
your own component, `MyAwesomeButton`, with the class `my-awesome-button` on it and use that instead of using `UiButton`.

```html
<template>
  <UiButton class="my-awesome-button">
    <slot/>
  <UiButton>
</template>

<script setup>
import { UiButton } from '@infermedica/component-library';
</script>

<style lang="scss">
.my-awesome-button {
  --button-background: #1471C9;
  --button-hover-background: #0F5496;
  --button-active-background: #0B3D6D;
}
</style>
```

## Atomic Design and CSS custom properties
To configure a nested component, you should use its CSS custom properties. e.g.
We don't create "proxy variables" for nested atoms. To configure a nested component, you should use its variables. e.g.

<Code code={`.ui-number-stepper {
  --button-icon-color: #D42E2E;
}`}/>

But `UiNumberStepper` contains two different buttons so you can use the class to override it.

<Code code={`.ui-number-stepper {
  &__decrement {
    --button-icon-color: #127F5B;
  }\n
  &__increment {
    --button-icon-color: #D42E2E;
  }
}`}/>

Or you can add custom classes to buttons by using props. e.g.

```html
<template>
  <UiNumberStepper
    :button-decrement-attrs="{class: 'my-awesome-decrement'}"
    :button-increment-attrs="{class: 'my-awesome-increment'}"
  />
</template>

<style lang="scss">
.my-awesome {
  &-decrement {
    --button-icon-color: #127F5B;
  }
  &-increment {
    --button-icon-color: #D42E2E;
  }
}
</style>
```

This is the recommended approach to configuring the CSS of a specific nested component as it's easier to maintain.
