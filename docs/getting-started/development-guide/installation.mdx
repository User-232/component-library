# Installation
The Infermedica Component Library provides a beautiful, configurable, and accessible library of basic and advanced components for faster system design and application development.
Below you'll find the information you need to add it to your project.

<PageOutline/>

## How to install
You can create a new project or integrate the Infermedica Component Library into an existing application. The installation process is largely the same for both cases. Choose the installation that best suits your project.

```bash
pnpm add @infermedica/component-library
```

## Usage default styles
The Infermedica Component Library is a highly configurable UI components library. To configure components to your needs, you can use a wide variety of global CSS variables and adjust their styles to your requirements. More details are provided in the <a href="/docs/getting-started-development-guide-theme-configuration--page">Theme Configuration</a> guidelines.

We provide our default designs. If you'd like to use them for your project, you need to import them into the `main.js` file.

```js
import '@infermedica/component-library/styles/styles.scss'
```

## Project config and loaders

### SVG Loader
We use a custom loader to load SVG files for the UiIcon component.

#### Webpack
For Webpack, use the [vue-svg-loader](https://www.npmjs.com/package/vue-svg-loader).
For Vue 3, you should use the latest version of vue-svg-loader.

```bash
pnpm i vue-svg-loader@next
````

To use the `vue-svg-loader`, you should add it to the webpack config. e.g. for vue-cli project.

```js
// vue.config.js
module.exports = {
  chainWebpack: (config) => {
    const svgRule = config.module.rule('svg');

    // clear all existing loaders.
    // if you don't do this, the loader below will be appended to
    // existing loaders of the rule.
    svgRule.uses.clear();

    // add replacement loader(s)
    svgRule
    .use('vue-loader')
    .loader('vue-loader')
    .end()
    .use('vue-svg-loader')
    .loader('vue-svg-loader')
    // options is optional, below you can find our svgo settings
    .options({
      svgo: {
        plugins: [
          { removeDimensions: true },
          { removeUselessStrokeAndFill: true },
          { convertStyleToAttrs: true },
        ]
      }
    })
  }
}
```

### Vite
For Vite, use the [vite-svg-loader](https://www.npmjs.com/package/vite-svg-loader).

```bash
pnpm i vite-svg-loader@next
````

To use the `vite-svg-loader`, you should add it to the vite config. e.g.

```js
// vite.config.loader
export default defineConfig({
  plugins: [
    vue(),
    svgLoader({
      svgoConfig: {
        plugins: [
          'removeDimensions',
          'removeUselessStrokeAndFill',
          'convertStyleToAttrs',
        ],
      },
    }),
  ]
})
```

## Usage components in SFC
You can also directly import components from the Infermedica Component Library and use them in a template.

### Script syntax

```html
<template>
  <UiComponentName>
</template>

<script>
import { UiComponentName } from '@infermedica/component-library'

export default {
  components: {
    UiComponentName
  }
}
</script>
```

### Script setup syntax

```html
<template>
  <UiComponentName/>
</template>

<script setup>
  import { UiComponentName } from '@infermedica/component-library'
</script>
```

### Internal components
Some components - like UiBulletPoints, UiTabs, etc. - use the `_internal` items component e.g. `UiBulletPoints` use `UiBulletPointsItem`.
To use this in a template, you should import it using the full path. e.g.

```html
<template>
  <UiToggleButtonGroup>
    <UiToggleButton/>
  </UiToggleButtonGroup>
</template>

<script setup>
  import { UiToggleButtonGroup } from '@infermedica/component-library'
  import UiToggleButton from '@infermedica/component-library/src/components/molecules/UiToggleButtonGroup/_internal/UiToggleButton.vue';
</script>
```

You can use a custom export and skip `src/components` from import.

```html
<script setup>
  import UiToggleButton from '@infermedica/component-library/molecules/UiToggleButtonGroup/_internal/UiToggleButton.vue';
</script>
```

For common cases, you should use the `items` props from the mentioned components.
